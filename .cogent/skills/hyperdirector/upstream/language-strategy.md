# Language Strategy

> HyperDirector v0.1 uses a three-layer language strategy.
> This file is the canonical reference for all language decisions.

---

## Three Layers

```
Layer 1: Engineering         → English only
Layer 2: User Delivery       → Chinese (complete, not partial)
Layer 3: Generated Content   → Follows brief.language (default: zh-CN for Chinese users)
```

---

## Layer 1: Engineering Layer (English)

All engineering artifacts use English exclusively. This ensures:
- Open-source collaboration without language barriers
- Clean diff history and code reviews
- Compatibility with Hermes, HyperFrames, and standard tooling
- Future PR readiness for contribution back to Hermes or HyperFrames

**Engineering layer includes:**

| Category | Files/Directories |
|---|---|
| Skill entry | `SKILL.md`, `AGENTS.md` |
| Core docs | `README.md` |
| Upstream rules | `upstream/` (all files) |
| Prompts | `prompts/` (control instructions in English) |
| Rules | `rules/` (all rule files) |
| Schemas | `schemas/` (all JSON schemas, field names in English) |
| Templates | `templates/` (template HTML, DESIGN.md, prompt.md) |
| Scripts | `scripts/` (all shell/JS scripts) |
| QA | `qa/` (pre-render-checklist.md, lint-fix-loop.md, final-delivery-checklist.md) |
| Output structure | `output/` (directory names in English) |

**Rules for engineering layer:**
- File names: kebab-case English only (e.g., `brand-kit.schema.json`, not `品牌规范.json`)
- Directory names: kebab-case English only
- JSON field names (keys): English camelCase or snake_case (e.g., `brand_name`, `motionLanguage`)
- Variable names in scripts: English
- Template IDs and composition IDs: English kebab-case
- CLI commands: English
- Git commit messages: English

---

## Layer 2: User Delivery Layer (Chinese — Complete)

Chinese user documentation must be complete and self-contained. Not a translation summary — full content.

**User delivery layer includes:**

| File | Purpose |
|---|---|
| `README.zh-CN.md` | Main Chinese entry point |
| `CAPABILITY_BOUNDARY.zh-CN.md` | Chinese capability boundary explanation |
| `docs/quickstart.zh-CN.md` | Quickstart guide |
| `docs/installation.zh-CN.md` | Installation and environment setup |
| `docs/first-video.zh-CN.md` | First video walkthrough |
| `docs/brand-kit-setup.zh-CN.md` | Brand Kit setup guide |
| `docs/template-guide.zh-CN.md` | Template selection guide |
| `docs/faq.zh-CN.md` | Frequently asked questions |
| `workflows/*.zh-CN.md` | Chinese workflow guides |
| `examples/zh-CN/` | Chinese demo projects |
| `brand/brand-kit.persona-zh.example.json` | Chinese persona-style brand kit example |

**Rules for user delivery layer:**
- File naming: `<name>.zh-CN.md` for Chinese, `<name>.md` for English
- Chinese documents must have real content — not placeholders or partial translations
- Chinese documents are maintained as primary, not generated from English
- JSON value fields (not keys) may be Chinese (e.g., `"brand_name": "示例创作者"`)
- The Chinese README must link to the English README and vice versa

**Quality bar:**
Chinese user documentation must be self-sufficient. A Chinese user should be able to install, configure, and run HyperDirector without reading any English documentation.

## Layer 3: Generated Content Layer (Follows brief.language)

Video content generated by HyperDirector follows the language specified in `brief.json`.

**Language fields:**

| Field | Location | Description |
|---|---|---|
| `brief.language` | `brief.json` | Script and narration language |
| `brief.subtitle_language` | `brief.json` | On-screen caption language |
| `brief.ui_copy_language` | `brief.json` | CTA and UI copy language |
| `brand_kit.locale` | `brand-kit.json` | Brand's default locale |
| `brand_kit.default_output_language` | `brand-kit.json` | Default video output language |
| `brand_kit.language_policy.script_language` | `brand-kit.json` | Script language override |
| `brand_kit.language_policy.subtitle_language` | `brand-kit.json` | Subtitle language override |
| `brand_kit.language_policy.ui_copy_language` | `brand-kit.json` | UI copy language override |
| `brand_kit.language_policy.technical_terms` | `brand-kit.json` | How to handle technical terms |

**Inference rules (when brief.language is absent):**
1. If user prompt is in Chinese → infer `zh-CN`
2. If user prompt is in English → infer `en-US`
3. If `brand-kit.default_output_language` is set → use that
4. Default fallback: `zh-CN` (primary audience)

**Chinese video default behavior:**
- Scripts: Chinese
- Captions: Chinese
- CTA text: Chinese
- On-screen headlines: Chinese
- Technical terms (API, GSAP, HyperFrames, etc.): keep English, add Chinese explanation if needed
- Voice: TTS with Chinese-compatible voice (af_heart or equivalent)

**Prompt engineering rule:**
Prompts in `prompts/` use English control instructions but must include the directive:
```
User-facing content (script, captions, headlines, CTA, on-screen copy) must be in the language 
specified by brief.language. If brief.language is "zh-CN", all user-facing content must be Chinese.
```

## File Naming Conventions

### Documents

```
<name>.md              → English primary document
<name>.zh-CN.md        → Chinese document (same topic, full content)
```

### Schemas

```
<entity>.schema.json   → JSON Schema definition (field names in English)
<entity>.example.json  → English example
<entity>.<brand>.example.json → Brand-specific example (e.g., brand-kit.persona-zh.example.json for OSS; real creator kits stay private)
```

### Templates

```
templates/<template-name>/    → Template directory (English kebab-case)
  template.html               → Main HTML template
  DESIGN.md                   → English design spec
  prompt.md                   → English generation prompt
  customization-guide.md      → English customization guide
```

### Workflows

```
workflows/<workflow-name>.md         → English workflow
workflows/<workflow-name>.zh-CN.md   → Chinese workflow
```

### Examples

```
examples/zh-CN/<project-name>/    → Chinese demo project
examples/en/<project-name>/       → English demo project (v0.2+)
```

### Output (generated per run)

```
output/<project-slug>/
  brief.json
  storyboard.json
  DESIGN.md
  index.html
  preview.html
  assets/
  render-report.md
  edit-instructions.md
  final.mp4             ← only if render was executed
```

## Summary

| Layer | Language | Rationale |
|---|---|---|
| Engineering | English | Open-source, tooling compatibility |
| User delivery docs | Chinese (complete) | Primary audience is Chinese creators |
| Generated video content | brief.language (default zh-CN) | Follows user's content language |
| JSON field names (keys) | English | Standard schema convention |
| JSON field values | May be Chinese | Content is user data |
| File/directory names | English kebab-case | Cross-platform, git-friendly |
| Prompt control instructions | English | LLM instruction clarity |
| Prompt content directives | Must specify brief.language | User-facing content follows brief |

**One-line rule:**
> Engineering speaks English, users receive Chinese, videos speak whatever language the brief says.
