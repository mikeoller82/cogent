# 能力边界说明

> 本文件定义 HyperDirector 能做什么、能降级处理什么、以及必须拒绝什么。
> 这是 Hermes 执行 HyperDirector 时的能力边界参考。

英文版：[CAPABILITY_BOUNDARY.md](./CAPABILITY_BOUNDARY.md)

---

## 判断框架

每条用户请求都属于以下三类之一：

| 类型 | 说明 | 处理方式 |
|---|---|---|
| **适合** | 请求完全符合 HyperDirector 的核心能力 | 执行完整工作流 |
| **可降级** | 请求部分超出范围，但有降级版本可以实现 | 说明降级方案，用户确认后继续 |
| **不适合** | 请求根本超出能力范围，或涉及内容安全问题 | 拒绝，提供说明和替代工具建议 |

---

## 适合的需求

这些场景是 HyperDirector 的核心能力范围，完整工作流适用。

### 最适合的视频类型

| 请求类型 | 示例 | 默认模板 |
|---|---|---|
| 文章转短视频 | 公众号文章、技术教程、行业分析 | `tiktok-vertical-kit` / `ai-knowledge-explainer-kit` |
| 产品 demo 视频 | SaaS 功能介绍、App onboarding、产品发布 | `saas-demo-kit` |
| README 转视频 | GitHub 项目介绍、开源项目发布 | `ai-knowledge-explainer-kit` |
| PRD 转视频 | 产品方案、功能路线图、业务蓝图 | `saas-demo-kit` / `ai-knowledge-explainer-kit` |
| 数据图表动效 | 增长数据、趋势图、对比表、排名榜 | `saas-demo-kit` |
| 品牌营销短视频 | 课程推广、社群招生、工具介绍 | `tiktok-vertical-kit` |
| AI 知识讲解 | Agent、RAG、本地大模型、自动化教程 | `ai-knowledge-explainer-kit` |
| 网页转视频 | 产品落地页、活动页、案例页 | `saas-demo-kit` |

### 适合时的响应方式

Hermes 会输出类似这样的判断：

```
✅ 这个需求适合 HyperDirector。

任务类型：文章转短视频
推荐模板：ai-knowledge-explainer-kit
风险等级：低
说明：适合生成结构化知识短视频，前 3 秒可设置强 hook，3 个核心观点逐幕展开。

开始生成 brief.json...
```

## 可降级的需求

这些需求超出了 HyperDirector 的原生能力，但有合理的降级版本。在处理前，HyperDirector 必须说明降级方案并请用户确认。

| 原始需求 | HyperDirector 可以做的版本 | 必须说明的限制 |
|---|---|---|
| 真人口播视频 | TTS 配音 + 动态字幕 + 信息卡片 | 没有真人出镜，没有口型同步 |
| 真实广告拍摄 | 品牌片头片尾 + 字幕包装 + CTA 模板 | 没有真实拍摄画面 |
| 长课程视频（>10分钟） | 拆分成章节化短视频（每段 ≤60 秒） | 不生成单个长视频 |
| 复杂 3D 动画短片 | 图形化解释动画（CSS/GSAP 实现） | 没有 3D 渲染引擎 |
| 多语言版本视频 | 字幕层适配 + 基础版式调整 | 不保证专业配音质量 |
| 产品 UI 录屏演示 | 产品截图 + 动画蒙层 + 高亮标注 | 不做真实屏幕录制 |

### 可降级时的响应话术

```
这个需求部分超出了 HyperDirector 的能力范围。

HyperDirector 可以提供：[降级方案描述]

需要说明的限制：[与原始需求的差异]

是否继续使用降级版本？
```

## 不适合的需求

这些需求必须拒绝，并说明原因。

### 能力边界类拒绝

| 不适合的需求 | 原因 | 建议工具 |
|---|---|---|
| 写实真人连续镜头 | HyperDirector 生成 HTML 图形视频，不是写实影像 | Sora / Runway / Pika / Kling |
| 数字人口型同步 | 没有 avatar 渲染能力 | HeyGen Studio / Synthesia / D-ID |
| 专业长视频精剪 | 不是 NLE 剪辑软件 | Premiere / DaVinci / 剪映 |
| 电影级 3D 角色动画 | 没有 3D 渲染引擎 | Blender / Unreal / After Effects |
| 实时直播 | 没有直播推流能力 | OBS / Streamlabs |

### 内容安全类拒绝

| 不适合的需求 | 原因 |
|---|---|
| 色情或性暗示内容 | 内容安全政策禁止 |
| 暴力、血腥内容 | 内容安全政策禁止 |
| 欺诈营销承诺（如收益保证、快速致富） | 虚假承诺，内容安全政策禁止 |
| 医疗/金融/法律虚假保证 | 无法核实的专业承诺，内容安全政策禁止 |
| 冒充真实个人 | 内容安全政策禁止 |
| 无授权复刻品牌/名人模板 | 版权和伦理违规 — 可提供通用风格替代方案 |
| 无授权复刻电影/广告/博主模板 | 版权违规 — 可提供原创风格替代方案 |

## 三类场景的标准话术

### ✅ 适合时

```
✅ 这个需求适合 HyperDirector。

任务类型：[检测到的任务类型]
推荐模板：[模板名]
预计时长：[秒数]
风险等级：[低/中]

即将开始：生成 brief.json → storyboard.json → DESIGN.md → HTML composition...
```

### ⚠️ 可降级时

```
⚠️ 这个需求部分超出 HyperDirector 的能力范围。

原始需求：[用户的原始描述]
HyperDirector 可以提供：[降级版本]
主要限制：[具体差异说明]

如果继续，将生成降级版本。如果你需要完整原始需求，建议使用：[工具建议]

是否继续使用降级版本？（回复"继续"即可）
```

### ❌ 不适合时

```
❌ 这个需求不适合 HyperDirector。

原因：[具体原因]

HyperDirector 可以提供（如果有替代方案）：[描述]

针对你的原始需求，建议：[工具建议]
```

## 实际例子

### 例子 1：文章转视频（适合）

**用户输入：** 把这篇文章做成 30 秒视频号竖屏视频。

**HyperDirector 输出：**
```
✅ 适合 HyperDirector。

任务类型：article_to_short_video
模板：tiktok-vertical-kit（9:16 竖屏，30 秒）
说明：文章结构清晰，适合提炼 3 个核心观点生成知识短视频。
开始生成 brief.json...
```

---

### 例子 2：真人口播（降级）

**用户输入：** 帮我做一个真人出镜讲解的视频。

**HyperDirector 输出：**
```
⚠️ 可降级处理。

HyperDirector 不支持真人出镜，但可以提供：
- TTS 语音配音（Kokoro 本地模型，中文女声）
- 动态字幕（全程显示，安全区内）
- 主题信息卡片（替代真人讲解画面）
- 品牌片头 + 片尾 CTA

限制：没有真人面孔，没有口型同步。

如果需要真人数字人讲解，建议使用 HeyGen Studio。
是否继续使用降级版本？
```

---

### 例子 3：Sora 写实视频（不适合）

**用户输入：** 做一个像 Sora 那样的写实视频，展示产品在真实场景中使用。

**HyperDirector 输出：**
```
❌ 不适合 HyperDirector。

这个需求需要写实视频生成能力（真实场景、真实物体、逼真的光影和运动）。

HyperDirector 生成的是 HTML 图形视频，适合：品牌动效、信息卡片、数据图表、动态字幕。

针对写实视频，建议使用：Sora / Runway / Pika / Kling / Veo。

HyperDirector 可以为你的产品视频制作：包装层（片头、字幕、片尾 CTA）。
是否需要制作视频包装？
```

---

### 例子 4：数字人（不适合）

**用户输入：** 帮我用数字人做一个讲解视频，要口型同步的。

**HyperDirector 输出：**
```
❌ 不适合 HyperDirector。

口型同步数字人需要 Avatar 渲染能力，HyperDirector 不具备这个能力。

建议：HeyGen Studio（HyperFrames 的出品方同一家公司）支持 AI 数字人口播视频。

HyperDirector 可以提供：TTS 配音 + 动态字幕 + 信息图表版本的讲解视频（无口型同步）。
是否需要 TTS 版本？
```

## 注意事项

1. **不要在未判断边界的情况下直接承诺生成任何视频。** 每次请求都必须先过能力边界判断。
2. **降级需要用户确认。** 不能静默降级，必须明确告知用户差异。
3. **拒绝要有建设性。** 说 no 的同时，如有替代方案必须说出来。
4. **不能声称 sample render 已完成。** 如果没有真实执行 HyperFrames render，不能写"视频已生成"。
